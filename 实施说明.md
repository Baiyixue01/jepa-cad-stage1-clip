# 实验说明：JEPA 方式训练 VLM 预测 CAD 操作高亮图 embedding

> 进行两个实验：
>
> ```text
> 实验 A：冻结 CLIP，只训练 fusion + predictor
> 实验 B：LoRA 微调 CLIP image encoder + text encoder，同时训练 fusion + predictor
> ```
>
> 核心任务保持不变：
>
> ```text
> 输入：before image + instruction
> 输出：highlight image 的 embedding
> 测试：预测 embedding 与 GT highlight embedding 的相似度 / retrieval 排名
> ```

---

## 1. 实验目标

训练一个 JEPA / VLM 风格模型：

```text
before_image + instruction
        ↓
CLIP image encoder + CLIP text encoder
        ↓
context fusion + predictor
        ↓
predicted_highlight_embedding
```

其中 GT 目标是：

```text
highlight_image
        ↓
CLIP image encoder / target encoder
        ↓
gt_highlight_embedding
```

训练目标：

```text
predicted_highlight_embedding 接近 gt_highlight_embedding
```

测试目标：

```text
对测试集每个样本：
    用 before_image + instruction 得到 predicted embedding
    用所有测试样本的 highlight image 得到 gt embedding 库
    计算相似度
    查看正确 highlight embedding 的排名
```

主要评估指标：

```text
Cosine Similarity
Top-1
Top-5
Top-10
MRR
Mean Rank
Median Rank
```

## 2. 两个实验设置

### 2.1 实验 A：冻结 CLIP，只训练 predictor

实验名：

```python
EXPERIMENT_NAME = "exp_a_freeze_clip_predictor"
```

训练策略：

```text
CLIP image encoder：冻结
CLIP text encoder：冻结
target CLIP image encoder：冻结
fusion MLP：训练
predictor：训练
```

结构：

```text
before_image
    ↓
frozen CLIP image encoder
    ↓
z_before

instruction
    ↓
frozen CLIP text encoder
    ↓
z_text

concat(z_before, z_text)
    ↓
trainable fusion MLP
    ↓
z_context
    ↓
trainable predictor
    ↓
z_pred_highlight

highlight_image
    ↓
frozen CLIP image encoder
    ↓
z_gt_highlight
```

这个实验的目的：

```text
验证在不微调 CLIP 的情况下，before image + instruction 是否已经能够预测 highlight embedding。
```

如果实验 A 的 retrieval 明显高于随机，说明该任务本身有学习信号。

---

### 2.2 实验 B：LoRA 微调 CLIP image encoder + text encoder

实验名：

```python
EXPERIMENT_NAME = "exp_b_lora_clip_image_text"
```

训练策略：

```text
CLIP image encoder：加入 LoRA，可训练 LoRA 参数
CLIP text encoder：加入 LoRA，可训练 LoRA 参数
target CLIP image encoder：建议冻结
fusion MLP：训练
predictor：训练
```

结构：

```text
before_image
    ↓
CLIP image encoder + LoRA
    ↓
z_before

instruction
    ↓
CLIP text encoder + LoRA
    ↓
z_text

concat(z_before, z_text)
    ↓
trainable fusion MLP
    ↓
z_context
    ↓
trainable predictor
    ↓
z_pred_highlight

highlight_image
    ↓
frozen target CLIP image encoder
    ↓
z_gt_highlight
```

这个实验的目的：

```text
验证对 CLIP 的 image/text encoder 做 CAD 领域适配后，是否能提升 highlight embedding 预测效果。
```

注意：

```text
实验 B 中 target encoder 建议仍然冻结。
```

原因：

```text
这样 gt_highlight_embedding 的目标空间稳定，训练更容易收敛。
```

## 9. model.py

实现 CLIP 版本的 JEPA 模型。

### 9.1 总体结构

```text
before image -> CLIP image encoder -> z_before
instruction -> CLIP text encoder -> z_text

concat(z_before, z_text)
        ↓
fusion MLP
        ↓
z_context
        ↓
predictor
        ↓
z_pred_highlight

highlight image -> target CLIP image encoder -> z_gt_highlight
```

---

### 9.2 实验 A 模型行为

实验 A：

```text
CLIP image encoder 冻结
CLIP text encoder 冻结
target CLIP image encoder 冻结
只训练 fusion + predictor
```

forward：

```python
z_before = clip.get_image_features(before_pixel_values)
z_text = clip.get_text_features(input_ids, attention_mask)

z_context = fusion(torch.cat([z_before, z_text], dim=-1))
z_pred = predictor(z_context)

with torch.no_grad():
    z_target = target_clip.get_image_features(highlight_pixel_values)
```

---

### 9.3 实验 B 模型行为

实验 B：

```text
CLIP image encoder 加 LoRA
CLIP text encoder 加 LoRA
target CLIP image encoder 冻结
训练 LoRA 参数 + fusion + predictor
```

forward 逻辑与实验 A 相同，但 `clip` 的 LoRA 参数参与训练。

---

### 9.4 Fusion + Predictor

```python
fusion = nn.Sequential(
    nn.Linear(EMBED_DIM * 2, EMBED_DIM),
    nn.GELU(),
    nn.LayerNorm(EMBED_DIM),
    nn.Linear(EMBED_DIM, EMBED_DIM)
)

predictor = nn.Sequential(
    nn.LayerNorm(EMBED_DIM),
    nn.Linear(EMBED_DIM, EMBED_DIM * 4),
    nn.GELU(),
    nn.Linear(EMBED_DIM * 4, EMBED_DIM)
)
```

forward 输出：

```python
{
    "z_pred": z_pred,
    "z_target": z_target
}
```

## 11. loss

训练使用三个 loss。

### 11.1 cosine loss

```python
loss_cos = 1 - cosine_similarity(z_pred, z_target.detach()).mean()
```

### 11.2 MSE loss

```python
loss_mse = mse_loss(z_pred, z_target.detach())
```

### 11.3 contrastive loss

在一个 batch 内：

```text
z_pred_i 应该最接近 z_target_i
```

计算：

```python
z_pred_norm = normalize(z_pred)
z_target_norm = normalize(z_target.detach())

logits = z_pred_norm @ z_target_norm.T / TEMPERATURE
labels = torch.arange(batch_size).to(device)

loss_contrastive = cross_entropy(logits, labels)
```

总 loss：

```python
loss = (
    LAMBDA_COS * loss_cos
    + LAMBDA_MSE * loss_mse
    + LAMBDA_CONTRASTIVE * loss_contrastive
)
```

## 14. eval_retrieval.py

功能：

```text
1. 指定实验目录，加载 best checkpoint。
2. 对 test set 每个样本计算 z_pred。
3. 对 test set 每个样本计算 z_target。
4. 计算相似度矩阵。
5. 统计检索指标。
6. 保存结果。
```

由于不使用 argparse，可以写两个入口：

```python
def eval_experiment(exp_cfg):
    ...
```

然后可以在文件底部执行：

```python
if __name__ == "__main__":
    from config import EXP_A, EXP_B
    eval_experiment(EXP_A)
    eval_experiment(EXP_B)
```

相似度矩阵：

```python
sim = normalize(z_pred) @ normalize(z_target).T
```

对于第 i 个样本，正确答案是第 i 个 target embedding。

指标：

```python
top1 = rank <= 1
top5 = rank <= 5
top10 = rank <= 10
mrr = mean(1 / rank)
mean_rank
median_rank
```

输出文件：

```text
{exp_output_dir}/eval/test_retrieval_summary.json
{exp_output_dir}/eval/test_retrieval_details.csv
{exp_output_dir}/eval/test_similarity_matrix.npy
```

## 15. 对比结果汇总

训练完实验 A 和实验 B 后，生成一个汇总文件：

```text
outputs/compare_exp_a_b.csv
```

字段：

```text
experiment_name
top1
top5
top10
mrr
mean_rank
median_rank
best_epoch
best_checkpoint
```

需要重点比较：

```text
实验 B 是否优于实验 A
```

如果：

```text
实验 B > 实验 A
```

说明 LoRA 微调 CLIP 对 CAD 高亮 embedding 预测有效。

如果：

```text
实验 B ≈ 实验 A
```

说明冻结 CLIP 已经足够，或者 LoRA 设置/学习率/数据质量需要调整。

如果：

```text
实验 B < 实验 A
```

说明 LoRA 可能破坏了原始 CLIP 表征，建议降低 LoRA lr 或只对 image encoder 加 LoRA。
